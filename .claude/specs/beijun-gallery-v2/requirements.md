# 贝军国画网站 V2 重构需求文档

## 介绍

本项目旨在将现有的贝军国画静态展示网站重构为一个现代化的全栈应用。新版本将采用 2025-2026 年主流技术架构（Next.js 15 App Router），在保留原有画作展示功能的基础上，新增**简洁高效的管理后台**，使画家能够方便地更新网站内容。

**设计理念：**
- **留白即是美** - 极简主义设计，让艺术作品成为唯一主角
- **沉浸式叙事** - 滚动触发的章节式体验，讲述画家的艺术故事
- **东方美学** - 融合传统水墨意境与现代数字交互
- **微动效点睛** - 精制的过渡动画，尊重 prefers-reduced-motion

**核心目标：**
1. 打造 2025 年顶级艺术家个人站点体验
2. 现代化技术栈升级（Next.js 15 SSG/ISR）
3. 简洁实用的内容管理后台
4. 极致的性能与 SEO 表现

---

## 需求

### 1. 访客端 - 沉浸式首页

**用户故事:** 作为访客，我希望通过首页获得沉浸式的艺术体验，感受画家的独特风格。

**验收标准:**

1.1. 系统应显示全屏 Hero 区域，采用视差滚动效果展示背景画作，配合优雅的标题动画

1.2. 系统应在滚动时展示"艺术理念"章节，使用渐入动画呈现画家的创作理念（1-2 句精炼文案）

1.3. 系统应展示 3 个精选画作，采用 Bento Grid 或错落布局，悬停时显示画作名称和年份

1.4. 当用户点击精选画作时，应平滑过渡到画廊页面并高亮该作品

1.5. 系统应在页面底部展示简约的画家签名/印章元素，增强东方美学氛围

1.6. 所有动画应尊重 `prefers-reduced-motion` 用户偏好，提供无动画降级体验

1.7. 页面应支持 SSG + ISR，内容更新后 60 秒内自动刷新

---

### 2. 访客端 - 画作画廊

**用户故事:** 作为访客，我希望优雅地浏览画家的全部作品，并能快速找到感兴趣的画作。

**验收标准:**

2.1. 系统应以 Masonry（瀑布流）布局展示画作，响应式适配 1/2/3/4 列

2.2. 画作卡片应采用极简设计：图片 + 悬停显示标题，无多余装饰

2.3. 系统应提供固定在顶部的筛选栏，包含：
   - 年份筛选（下拉，动态生成选项）
   - 状态筛选（全部/售卖中/可定制/已售出）
   - 实时搜索框（带 300ms 防抖）

2.4. 系统应支持无限滚动加载，或提供优雅的分页导航

2.5. 当用户点击画作时，系统应显示全屏 Lightbox 模态框：
   - 画作大图（支持缩放手势）
   - 标题、年份、尺寸、状态、描述
   - "咨询购买"按钮

2.6. Lightbox 应支持键盘导航（ESC 关闭，左右箭头切换作品）

2.7. "咨询购买"应显示联系方式（微信号 + 一键复制功能）

2.8. 筛选状态应同步到 URL，支持分享直达链接

---

### 3. 访客端 - 画家介绍

**用户故事:** 作为访客，我希望深入了解画家的背景与艺术历程。

**验收标准:**

3.1. 系统应展示画家大幅肖像，配合渐入动画

3.2. 系统应展示画家简介，支持富文本格式（从后台获取）

3.3. 系统应以时间线或章节形式展示艺术历程/成就

3.4. 页面底部应提供"浏览作品"CTA 链接到画廊

---

### 4. 访客端 - 交互创新

**用户故事:** 作为访客，我希望体验到 2025 年最前沿的网页交互设计。

**验收标准:**

4.1. 系统应实现平滑的页面过渡动画（View Transitions API 或 Framer Motion）

4.2. 系统应支持深色/浅色模式，默认跟随系统，可手动切换

4.3. 导航栏应采用"隐形导航"设计：
   - 首页时完全透明，融入 Hero 背景
   - 滚动后优雅显现（毛玻璃效果）
   - 滚动方向感知：向下滚动隐藏，向上滚动显示

4.4. 系统应实现滚动进度指示器（如页面边缘的细线进度条）

4.5. 画作 Lightbox 应支持拖拽关闭手势（移动端）

4.6. 加载状态应使用优雅的骨架屏或水墨晕染效果

---

### 5. 管理后台 - 简洁认证

**用户故事:** 作为画家，我希望通过简单的登录进入后台管理内容。

**验收标准:**

5.1. 系统应提供 /admin 登录页面，设计风格与前台一致

5.2. 系统应使用用户名/密码进行身份验证（单用户即可）

5.3. 登录成功后应重定向到后台首页

5.4. 会话有效期 7 天，过期后重新登录

5.5. 所有后台 API 应验证身份，未认证返回 401

---

### 6. 管理后台 - 首页内容管理

**用户故事:** 作为画家，我希望能够更新首页的展示内容。

**验收标准:**

6.1. 系统应提供 Hero 区域编辑：
   - 上传/更换背景图片（拖放上传，最大 5MB）
   - 编辑主标题、副标题

6.2. 系统应提供精选画作选择器：
   - 从现有画作中选择 3 幅
   - 支持拖拽排序

6.3. 保存后应触发 ISR 重新验证

---

### 7. 管理后台 - 画作管理（简化版）

**用户故事:** 作为画家，我希望能够管理画廊中的作品信息。

**验收标准:**

7.1. 系统应提供画作列表视图：缩略图 + 标题 + 年份 + 状态 + 上架状态

7.2. 系统应支持以下操作：
   - **上架/下架切换** - 控制画作是否在前台显示
   - **编辑基本信息** - 标题（必填）、描述（选填）
   - **选择创作年份** - 下拉选择 2000-当前年份
   - **设置售卖状态** - 枚举：售卖中/可定制/已售出

7.3. 系统应支持新增画作：上传图片 + 填写上述信息

7.4. 系统应支持删除画作（需确认）

7.5. 列表应支持按年份、状态、上架状态筛选

7.6. 图片上传应自动压缩并存储到 CDN

---

### 8. 管理后台 - 画家信息管理

**用户故事:** 作为画家，我希望能够更新自己的介绍信息。

**验收标准:**

8.1. 系统应提供画家信息编辑：
   - 上传头像
   - 编辑姓名
   - 编辑介绍（支持简单富文本）

8.2. 保存后应触发相关页面重新生成

---

### 9. 性能与 SEO

**用户故事:** 作为网站所有者，我希望网站加载快速且搜索引擎友好。

**验收标准:**

9.1. 首页 LCP < 2.5s，INP < 200ms，CLS < 0.1

9.2. 系统应为每个页面生成正确的 meta 标签

9.3. 系统应生成 sitemap.xml 和 robots.txt

9.4. 图片应使用 next/image 自动优化

9.5. 首屏 JS Bundle < 100KB (gzipped)

---

### 10. 可访问性

**用户故事:** 作为访客，我希望无论使用何种设备都能顺利浏览。

**验收标准:**

10.1. 可交互元素应有清晰的焦点状态

10.2. 图片应有描述性 alt 文本

10.3. 颜色对比度符合 WCAG AA 标准

10.4. 尊重 prefers-reduced-motion 用户偏好

10.5. 支持键盘完整导航

---

## 技术栈决策

| 类别 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js 15 App Router | RSC、Server Actions、ISR 支持 |
| 样式 | Tailwind CSS 4.0 + shadcn/ui | 极简主义设计、组件一致性 |
| 动画 | Framer Motion + View Transitions | 流畅的页面过渡和微交互 |
| 数据 | JSON 文件 + Server Actions | 轻量级、无需数据库 |
| 图片 | Vercel Blob / Cloudinary | 自动优化、CDN 分发 |
| 认证 | NextAuth.js Credentials | 单用户场景足够 |
| 部署 | Vercel | 零配置、Edge 全球加速 |

## 设计规范

### 色彩系统

| 用途 | 浅色模式 | 深色模式 |
|------|---------|---------|
| 背景 | #FAFAF9 (Stone-50) | #1C1917 (Stone-900) |
| 文字 | #0C0A09 (Stone-950) | #FAFAF9 (Stone-50) |
| 强调 | #CA8A04 (Amber-600) | #FCD34D (Amber-300) |
| 边框 | #D6D3D1 (Stone-300) | #44403C (Stone-700) |

### 字体

- 标题：Noto Serif SC（传统衬线）
- 正文：Noto Sans SC（现代无衬线）
- 英文辅助：Inter

### 间距

- 采用 8px 基准网格
- 大留白：营造水墨画般的空灵感

## 术语表

| 术语 | 定义 |
|------|------|
| Hero | 首页顶部全屏横幅区域 |
| Lightbox | 画作详情全屏弹窗 |
| ISR | 增量静态再生成，按需更新静态页面 |
| Bento Grid | 不规则网格布局，类似日式便当盒 |
| View Transitions | 浏览器原生页面过渡动画 API |
